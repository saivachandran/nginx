What Is a Reverse Proxy?
#

A reverse proxy is an intermediary proxy service which takes a client request, passes it on to one or more servers, and subsequently delivers the serverâ€™s response to the client. A common reverse proxy configuring is to put Nginx in front of an Apache web server. Using this method will allow both web servers to work together enabling each to do what they do best. Learn more about the differences between Nginx vs Apache.

This post will outline the benefits of using an Nginx reverse proxy as well as how to configure one.

How To Setup an Nginx Reverse proxy

After vhost installation 


step 1

edit /etc/nginx/sites-available/kavi.sai.com.conf  using vim add below lines at end of the file


server {
    listen kavi.sai.com:80;
    server_name  kavi.sai.com;
    location / {
        root /var/www/kavi.sai.com/html;
        proxy_set_header X-Forwarded-Host $host:$server_port;
        proxy_set_header X-Forwarded-Server $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_pass http://kavi.sai.com:81;
    }
}
